## User and Group Management Linux

## Types of Linux Users
```
1. Root User
The root user is the superuser in Linux.
Has unrestricted access to the entire system.
Can perform all administrative tasks, including:
 Creating and deleting users
 Modifying system files
 Installing software
 Configuring system settings
 User ID (UID): 0
```

```
Common Root Commands
sudo -i → Switch to root user
sudo <command> → Execute a command with root privileges
```


2. System Users
```   
System users are service accounts created by the system.
Used to run specific services or processes (e.g., web server, database).
Do not have login shells by default.
Have limited privileges to enhance system security.
User ID (UID) range: 1 – 999
```
---
```
4. Standard / Local Users
These users are created for daily tasks and normal operations.
Each user has:
Their own home directory (e.g., /home/username)
Specific permissions to access files and resources
Used for regular login and work.
User ID (UID) range: 1000 – 60000 (commonly)
```

## Linux user management commands
| No. | Task / Purpose             | Command Example         | Description                                                                    |
| --- | -------------------------- | ----------------------- | ------------------------------------------------------------------------------ |
| 1   | Add new user (interactive) | `adduser <username>`    | Creates user with home directory, group, password prompt, and default settings |
| 2   | Add new user (basic)       | `useradd <username>`    | Creates user but **does NOT create home directory by default**                 |
| 3   | Create user with home dir  | `useradd -m <username>` | Creates user along with home directory                                         |
| 4   | Set / change password      | `passwd <username>`     | Assigns or updates user password                                               |
| 5   | Delete user                | `userdel <username>`    | Removes user but keeps home directory                                          |
| 6   | Delete user + home dir     | `userdel -r <username>` | Removes user along with home directory and mail spool                          |
| 7   | List all users             | `cat /etc/passwd`       | Displays all system users                                                      |



```
/etc/passwd
```
---
1. Stores user account information

2. File is readable by all users


Example of /etc/passwd file =

### abhi:x:1001:1001:Abhi Kumar:/home/abhi:/bin/bash


## Fields in /etc/passwd (Fields separated by :)
| Field Name               | Value        | Description                             |
| ------------------------ | ------------ | --------------------------------------- |
| **Username**             | `abhi`       | Name of the user account                |
| **Password Placeholder** | `x`          | Actual password stored in `/etc/shadow` |
| **User ID (UID)**        | `1001`       | Unique ID of the user                   |
| **Group ID (GID)**       | `1001`       | Primary group ID of the user            |
| **User Info (GECOS)**    | `Abhi Kumar` | Full name / description of user         |
| **Home Directory**       | `/home/abhi` | Default login directory                 |
| **Login Shell**          | `/bin/bash`  | Shell assigned to the user              |



## 📂 /etc/shadow
 
 Stores encrypted passwords hashes

 Only accessible by root user
 
 Improves system security
 
 Also stores password aging information
 
 Improves system security

Example of /etc/shadow
## abhi:$6$Tz3...Qw/:19540:7:90:10:14:19999

 Fields in /etc/shadow (Fields separated by :)

| Field Name               | Value          | Description                                                   |
| ------------------------ | -------------- | ------------------------------------------------------------- |
| **Username**             | `abhi`         | Name of the user                                              |
| **Encrypted Password**   | `$6$Tz3...Qw/` | Encrypted (hashed) password (actual password is not readable) |
| **Last Password Change** | `19540`        | Number of days since **01-01-1970**                           |
| **Minimum Days**         | `7`            | Minimum days before password can be changed                   |
| **Maximum Days**         | `90`           | Password validity period                                      |
| **Warning Days**         | `10`           | Days before password expiry warning                           |
| **Inactive Days**        | `14`           | Days after expiry before account is locked                    |
| **Account Expiry Date**  | `19999`        | Date when account expires (`-1` or empty → never expires)     |





## Change Password Policy

chage Command 

chage In Linux is used the chage (change-age) command to manage and view password aging and

account expiration policies for user


## chage password aging command
| No. | Purpose                              | Command                          | Description                                                |
| --- | ------------------------------------ | -------------------------------- | ---------------------------------------------------------- |
| 1   | List password aging information      | `chage -l <username>`            | Displays current password expiry details for the user      |
| 2   | Set maximum password validity (days) | `chage -M 30 <username>`         | Password expires after 30 days                             |
| 3   | Set minimum days between changes     | `chage -m 7 <username>`          | User must wait 7 days before changing password again       |
| 4   | Set warning days before expiry       | `chage -W 10 <username>`         | User gets warning 10 days before password expires          |
| 5   | Set inactive days after expiry       | `chage -I 14 <username>`         | Account gets disabled if not used for 14 days after expiry |
| 6   | Set account expiry date              | `chage -E 2025-12-31 <username>` | Account will expire on this date                           |
| 7   | Force password change at next login  | `chage -d 0 <username>`          | Forces user to change password on next login               |


## Modify Existing User
usermod Command: Used to modify user account details
| No. | Purpose                  | Command Example                         | Description                                |
| --- | ------------------------ | --------------------------------------- | ------------------------------------------ |
| 1   | Assign no-login shell    | `usermod -s /sbin/nologin steve`        | Prevents user from logging into the system |
| 2   | Check current user shell | `echo $SHELL`                           | Displays current shell of logged-in user   |
| 3   | Check valid login shells | `cat /etc/shells`                       | Lists all allowed shells on the system     |
| 4   | Change User ID (UID)     | `usermod -u <UID> <username>`           | Modifies the user’s UID                    |
| 5   | Example: Change UID      | `usermod -u 3000 tony`                  | Sets UID of user `tony` to 3000            |
| 6   | Change home directory    | `usermod -d <new_home_dir> <username>`  | Updates user’s home directory path         |
| 7   | Add/modify user comment  | `usermod -c "development center" bruce` | Adds description (GECOS field)             |
| 8   | Lock user account        | `usermod -L <username>`                 | Disables user login (locks password)       |
| 9   | Unlock user account      | `usermod -U <username>`                 | Enables user login again                   |

## Sudoers File
What is Sudoers File?

/etc/sudoers is a configuration file

Controls who can run commands with elevated (root) privileges

Allows users to execute commands as root or superuser

Provides a secure way to manage administrative rights

## Add User to Sudoers File
```
1.Login as root

2.Open sudoers file:
vim /etc/sudoers

3.Go to the bottom of the file and add:

username ALL=(ALL) ALL

Meaning:
1st ALL → From any terminal
(ALL) → As any user

2nd ALL → Run all commands

```
##Important Note

Always edit sudoers file using : visudo (to avoid syntax errors)


## Group Management In Linux)

Group
A group is a collection of users that helps in managing permissions easily.


Example of cat /etc/group 

### developers:x:1005:alice,bob

## Fields in /etc/group (Fields separated by :)
| Field Name               | Value        | Description                                       |
| ------------------------ | ------------ | ------------------------------------------------- |
| **Group Name**           | `developers` | Name of the group                                 |
| **Password Placeholder** | `x`          | Actual group password is stored in `/etc/gshadow` |
| **Group ID (GID)**       | `1005`       | Numerical ID for the group                        |
| **Group Members**        | `alice,bob`  | Comma-separated list of users                     |






## Fields in /etc/gshadow

| Field                    | Description                    |
| ------------------------ | ------------------------------ |
| **Group Name**           | Name of the group              |
| **Encrypted Password**   | Stored group password (if any) |
|                          | `!` or `*` → No group password |
| **Group Administrators** | Users who can manage the group |
| **Group Members**        | Members of the group           |


## Commands for Group Management
| No. | Task / Purpose              | Command Example                 | Description                        |
| --- | --------------------------- | ------------------------------- | ---------------------------------- |
| 1   | Create a new group          | `groupadd devops`               | Creates a new group named `devops` |
| 2   | Delete a group              | `groupdel devops`               | Removes the group                  |
| 3   | Modify group name           | `groupmod -n newname oldname`   | Renames a group                    |
| 4   | Add user to a group         | `usermod -aG devops john`       | Adds user `john` to `devops` group |
| 5   | Add user to multiple groups | `usermod -aG devops,admin john` | Adds user to multiple groups       |
| 6   | Change primary group        | `usermod -g devops john`        | Sets primary group of user         |
| 7   | Remove user from group      | `gpasswd -d john devops`        | Removes user from group            |
| 8   | View user groups            | `groups john`                   | Shows groups of a user             |
| 9   | View group details          | `getent group devops`           | Displays group info                |
| 10  | List all groups             | `cat /etc/group`                | Shows all groups in system         |
| 11  | Set group password          | `gpasswd devops`                | Sets password for group            |
| 12  | Add group admin             | `gpasswd -A john devops`        | Assigns admin to group             |
| 13  | Add multiple users to group | `gpasswd -M user1,user2 devops` | Adds multiple users at once        |
